使用 VuePress 和 GitHub Pages 搭建个人博客

# 什么是 VuePress ？

VuePress 由两部分组成：一部分是支持用 Vue 开发主题的极简静态网站生成器，另一个部分是为书写技术文档而优化的默认主题。它的诞生初衷是为了支持 Vue 及其子项目的文档需求。

简单来说就是我们可以用它非常方便的生成简单的文档类型的静态页面。在 GitHub VuePress 有 13K+ ⭐️ ，也有开发者帮我们搭建好了一些基于 VuePress 的 Themes 和 Plugins 。今天我使用的就是其中一个 Theme vuepress-theme-meteorlxy 。

# 搭建前准备

  1. 确保你的开发环境安装了 Node.js 同时版本号 >= 8
#查看 Node.js 版本号
node -v
  1. 确保你有一个 GitHub 的账号, 并在开发环境装有 Git ，且配置好对应的 SSH Key
#查看 Git 版本号
git --version

  1. 确保在你的 GitHub 上有一个可以存放博客页面的 Repository ，且配置好对应的 Deploy Key

  2. 尽可能阅读一下 VuePress 官方上手指南，不阅读不影响本次搭建

# 搭建你的项目

# 新建项目、安装

这里我使用的所 npm ，也可以使用 Yarn 。

#新建一个路径作为项目的根路径
mkdir vuepress-blog

#cd 进去
cd vuepress-blog

#初始化一个 node 项目
npm init

#安装 VuePress vuepress-theme-meteorlxy
npm install -D vuepress vuepress-theme-meteorlxy

官方提醒，如果你的现有项目依赖了 webpack 3.x，推荐使用 Yarn 而不是 npm 来安装 VuePress。因为在这种情形下，npm 会生成错误的依赖树。

建立好如下的路径和文件，主题官方指南中使用 src 而不是 docs ，因为 GitHub Pages 要求使用 docs 路径，这里使用 docs ，两者没有实质差别。

vuepress-blog
├── docs # Blog 源文件目录
│   ├── .vuepress # Vuepress 目录
│   │   └── config.js # Vuepress 配置文件
│   └── _posts # 博客文件夹
│       ├── xxx.md
│       ...
└── package.json

注意，docs/index.md 或 docs/README.md 不是必须的，本主题会自动为你加上首页。

在 package.json 中加入 scripts 字段，在开发调试以及项目构建时会用到

{
  "scripts": {
    "docs:dev": "vuepress dev docs",
    "docs:build": "vuepress build docs"
  }
}

# 写第一篇博客

在 _posts 下新建一个 markdown 文件，写入下面的内容

<!-- _posts/2019-06-22-hello-world.md -->

---
category: hello
tags:
  - world
date: 2019-06-23
title: Hello, world!
vssue-title: Hello, world!
---

这是第一篇博文。

more 上面的内容是摘要，将显示在目录中。

<!-- more -->

more 下面的内容只有浏览这篇文章时才会完全展示，不会显示在目录中。

# 调试运行

运行前文配置的 scripts

npm run docs:dev

等待执行完毕后，在浏览器中输入 http://localhost:8080 即可查看到你的博客

此时，你可以尝试修改 config.js 的配置信息，比如 SNS 信息，以及你的 _posts 路径下的 markdown 文件，观察网页的变化。

# 使用评论功能

本主题通过 Vssue 启用页面评论功能。

# 配置评论

只有在 _posts 下的文章可以使用评论功能，其他地方不能使用 默认使用 Github 平台来存储评论，可以查看 Vssue 官方文档 了解如何配置 config.js 中 comments 各个参数，此时需要使用到前面提前准备到的 Repository ：

module.exports = {
  // 主题配置
  themeConfig: {
    // 评论配置
    comments: {
      platform: 'github', // 可选，默认使用 'github'，还可以选择 'gitlab', 'bitbucket'。详情参考 Vssue 文档
      owner: 'meteorlxy', // GitHub 的名字
      repo: 'vuepress-theme-meteorlxy', // Repository 名字
      clientId: 'MY_CLIENT_ID', // 在 GitHub 配置 Vssue 后给出
      clientSecret: 'MY_CLIENT_SECRET', // 在 GitHub 配置 Vssue 后给出
      autoCreateIssue: process.env.NODE_ENV !== 'development', // 可选，这样设置可以在开发环境下不自动创建 Issue
    },
  },
}

配置完成后，博客评论将储存在你的 Github 仓库的 Issue 系统中，每篇博文会自动创建一个 Issue，默认使用博文中的标题 title 作为 Issue 的标题，也可以自定义 vssue-title。 Vssue 默认根据 Vuepress 的 $lang 参数来确定使用的语言。

# 针对每篇文章单独配置

如果你博文的标题可能会有变动，建议你在 frontmatter 中加入一条 vssue-title 来作为 Issue 固定的标题，避免因为后续改动博文标题而引发问题：

---
category: hello
tags:
  - world
date: 2019-06-23
title: Hello, world!
vssue-title: 固定的 Issue 标题
---

这是第一篇博文。

如果你想要手动创建 Issue，那么你需要在 frontmatter 中设置 vssue-id，来对应 Issue 的 ID ：

---
category: hello
tags:
  - world
date: 2019-06-23
title: Hello, world!
vssue-id: 1
---

这是第一篇博文。

如果你想要关闭某篇博文下的评论，在 frontmatter 中设置 vssue: false 即可：

---
category: hello
tags:
  - world
date: 2019-01-21
title: Hello, world!
vssue: false
---

这是第一篇博文。

# 完全禁用评论

当然，如果你不想让你的博客拥有评论功能，你可以完全禁用它：

module.exports = {
  // 主题配置
  themeConfig: {
    // 评论配置
    comments: false
  },
}

# 部署到 GitHub Pages

GitHub Pages 可以在你的每一个分支分配一个 URL ，将页面放在 docs 并将页面 push 到该分支后即可访问到。

# 项目设置

在 docs/.vuepress/config.js 中设置正确的 base 。

 base:'/<REPO>/',

如果你打算发布到 https://<USERNAME>.github.io/ ，则可以省略这一步，因为 base 默认即是 "/" 。
如果你打算发布到 https://<USERNAME>.github.io/<REPO>/ （也就是说你的仓库在 https://github.com/<USERNAME>/<REPO>） ，则将 base 设置为 "/<REPO>/" 。

# 写发布脚本

在你的项目根路径中，创建一个如下的 deploy.sh 脚本（请自行判断去掉注释）:

#!/usr/bin/env sh

# 确保脚本抛出遇到的错误
set -e

# 生成静态文件
npm run docs:build

# 进入生成的文件夹
cd docs/.vuepress/dist

# 如果是发布到自定义域名
# echo 'www.example.com' > CNAME

git init
git add -A
git commit -m 'deploy'

# 如果发布到 https://<USERNAME>.github.io
# git push -f git@github.com:<USERNAME>/<USERNAME>.github.io.git master

# 如果发布到 https://<USERNAME>.github.io/<REPO>
# git push -f git@github.com:<USERNAME>/<REPO>.git master:gh-pages

cd -

# 执行脚本

sh deploy.sh |tee deploy.log

# 浏览器查看结果
